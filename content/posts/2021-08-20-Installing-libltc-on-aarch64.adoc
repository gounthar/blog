== Installing libltc on aarch64 Debian

One of my (way too many) current projects is to create a device that would mimic link:https://dish.tc/faq/[dish.tc].
I have created a link:https://gitlab.com/gounthar/time-whisperer[repository] at Gitlab so that anyone can help make this idea come true.

To sum up, we would add an audio track to every recording we make (be it audio or video or both) so that link:https://www.youtube.com/watch?v=9qjrMeCW1-c[video editing software] would then be able to synchronize everything without any human intervention.

I'm not sure yet of the SBC I will use for that job. For sure, I would like to use the link:http://www.orangepi.org/orangepizero/[OrangePi Zero], as it is link:https://www.aliexpress.com/item/1005001688034180.html?spm=a2g0o.store_pc_groupList.8148356.5.21c594fblAa9xq[cheap] (well, it used to be $10, but with the chip crisis, the price has gone wayyy up), or any other SBC, as long as it is handled by link:https://www.armbian.com/download/?device_support=Supported[Armbian] and has good audio features (yes, I know the Zero doesn't have great link:https://www.aliexpress.com/item/32770665186.html?spm=a2g0o.store_pc_groupList.8148356.1.7cad7fdd4Eu7Uf[audio features] nor link:https://forum.armbian.com/topic/6021-orange-pi-zero-i2s/?do=findComment&comment=52342[I2S], but I already have it, so...).

For the time being, I have a very well working aarch64 board, the link:http://bruno.verachten.fr/2021/02/18/How-to-Install-Armbian-on-the-Station-P1/[Station P1]. The sound is good (to me at least), so let's give it a try.

The first step to make a link:https://en.wikipedia.org/wiki/SMPTE_timecode[SMTPE] generator with this device is to install link:https://github.com/x42/libltc[libltc] which is not yet part of Debian for ARM.

== Grabbing the source, compiling and installing

=== Getting the source and configuring for the target machine

[source,bash]
----
git clone https://github.com/x42/libltc.git
cd libltc/
aclocal; autoheader; libtoolize --copy; autoconf; automake --gnu --add-missing --copy
./configure

----

== Packaging the official Debian version, but for aarch64

In order to build a binary package from source, first we need to  install necessary build tools. Depending upon the command line tool to  build the binary package, you may need to install any one of the  following packages:

. dpkg-dev
. devscripts

These packages provides numerous tools to build binary packages from source code.

To install "dpkg-dev", run:

----
$ sudo apt install dpkg-dev

----

To Install "devscripts" package, the command would be:

----
$ sudo apt install devscripts

----

=== Enable source repositories

Make sure you have enabled the source repositories.

To do so, open _/etc/apt/sources.list_ file:

----
$ sudo vim /etc/apt/sources.list

----

You will see some lines in it like below:

----
== See http://help.ubuntu.com/community/UpgradeNotes for how to upgrade to
== newer versions of the distribution.
deb http://in.archive.ubuntu.com/ubuntu focal main restricted
== deb-src http://in.archive.ubuntu.com/ubuntu focal main restricted
[...]

----

In the above file, all the lines beginning with one or  two hashes (#) are just comments, for information only. And the lines  without hashes are the _apt repositories_.

=== Install build dependencies for the package

Next, we need to install the dependencies for that package as well. To do so, run:

----
$ sudo apt build-dep libltc11

----

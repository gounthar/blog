== miniJen
The Jenkins multi-architecture CPU instance

image::/media/images/2023/02/16/fosdem_2023_booth_display.jpg[miniJen as a FOSDEM display on the booth, width=839]

== What is that contraption?

Nope, it’s not a robot of some sort, it won’t move by itself.
It’s not link:https://en.wikipedia.org/wiki/Cerebro[Cerebro] from Professor Xavier; no, it can’t link:https://all3dp.com/2/raspbery-pi-drone-simply-explained/[fly] either.
What you’re looking at is a Jenkins “link:https://www.jenkins.io/doc/book/installing/kubernetes/[cluster]” or “link:https://www.jenkins.io/doc/#what-is-jenkins[instance]”.
It is composed of a link:https://www.jenkins.io/doc/book/using/using-agents/#using-jenkins-agents[controller] (the “brain” or conductor) and three link:https://www.jenkins.io/doc/book/using/using-agents/[agents] (the workers or musicians if we continue a little further with the metaphor).

During link:https://fosdem.org/2023/[FOSDEM], we displayed the `aarch64` Jenkins controller dashboard on an another computer screen using the same Wi-Fi network.

This Jenkins instance was featured in the link:https://hackaday.com/2023/03/01/fosdem-2023-an-open-source-conference-literally/[Hackaday blog post] about FOSDEM.

These boards are not link:https://en.wikipedia.org/wiki/Microcontroller[microcontrollers], they are miniature computers running link:https://en.wikipedia.org/wiki/Linux[GNU/Linux], like the infamous link:https://en.wikipedia.org/wiki/Raspberry_Pi[Raspberry Pi].

== Hardware

The controller runs on a link:https://www.friendlyelec.com/index.php?route=product/product&product_id=287[NanoPi R5S], sold as a link:https://en.wikipedia.org/wiki/Router_(computing[router]) (thus the three link:https://en.wikipedia.org/wiki/Modular_connector#8P8C[RJ45] connectors).

image::/media/images/2023/02/16/NanoPi_R5S-01B.png[NanoPi R5S pic from the manufacturer, width=500]

It’s a 4GB link:https://en.wikipedia.org/wiki/AArch64[`aarch64`] (or `armv8`) link:https://wiki.friendlyelec.com/wiki/index.php/File:Rockchip_RK3568B2_Datasheet_V1.0.pdf[4 cores] running link:https://wiki.friendlyelec.com/wiki/index.php/FriendlyCore_(based_on_ubuntu-core_with_Qt[friendlyCore]), a distribution from the manufacturer (link:https://friendlyelec.com/[friendlyElec]) on a link:https://cdn.kernel.org/pub/linux/kernel/v5.x/ChangeLog-5.10[5.10.x kernel].

The smallest board is a 4 cores link:https://en.wikipedia.org/wiki/ARM_architecture_family#32-bit_architecture[`arm32`] agent with 512MB of RAM running link:https://en.wikipedia.org/wiki/Armbian[Armbian] with a 5.10.x kernel too.

image::/media/images/2023/02/16/NanoPi_Duo2-2.png[NanoPi Duo2 pic from the manufacturer, width=500]

It's also a board coming from the link:https://friendlyelec.com/[friendlyElec] manufacturer, the link:https://www.friendlyelec.com/index.php?route=product/product&path=69&product_id=244&sort=p.price&order=ASC[NanoPi Duo2].

The pink board next to the `arm32` board is a link:https://en.wikipedia.org/wiki/RISC-V[`RISC-V`] board running link:https://forum.armbian.com/topic/21465-armbian-image-and-build-support-for-risc-v/[Armbian] with just 1 core, 1GB of RAM and a link:https://cdn.kernel.org/pub/linux/kernel/v6.x/ChangeLog-6.1[6.1.x kernel].

image::/media/images/2023/02/23/mq-pro.png[MangoPi MQ-Pro pic from the manufacturer, width=500]

It's a MangoPi link:https://mangopi.org/mqpro[MQ-Pro] from link:https://mangopi.org/[MangoPi], one of the first `RISC-V` boards available.

The latest board just next to the `RISC-V` board with a slightly different shade of pink is an `aarch64` board also from link:https://mangopi.org/[MangoPi].

[image::/media/images/2023/02/16/mq-quad.png[MangoPi MQ-Quad pic from a taobao store, width=500]](https://world.taobao.com/item/662901818090.htm)

It is a 4 cores agent with 1GB of RAM running a fork of Armbian with link:https://cdn.kernel.org/pub/linux/kernel/v5.x/ChangeLog-5.16[kernel 5.16.x].
It's a link:https://mangopi.org/mqquad[MangoPi MQ-Quad].

== Don't try to fool me, there are no cables between the boards!

The boards all have Wi-Fi, and they are all connected to the same Wi-Fi network, provided by a router or my phone, depending on the location.
You can spot their small Wi-Fi antennas hanging in the first pic, except for the router which has no integrated Wi-Fi (it uses a link:https://www.realtek.com/en/products/communications-network-ics/item/rtl8821cu[USB Wi-Fi dongle] you can see in the pic).
One day, the R5S controller will also be a router for miniJen, but for now, it’s just a Jenkins controller.
How come the controller can contact and control the agents? We're not using IP addresses, but hostnames ending in `.local`, thanks to the link:https://en.wikipedia.org/wiki/Avahi_(software[Avahi]) daemon.

== What is that big box with cables?

image::/media/images/2023/02/16/pine-power.png[PinePower powering very astemious boards, width=839]

These boards are powered thanks to a link:https://www.pine64.org/[Pine64] link:https://www.pine64.org/pinepowerdesktop/[power supply].
Most of the time, you can see they don’t use much current.

== 3D printed parts

image::/media/images/2023/02/16/round-booth-display-2023-02-02-transparent.png[The 3DDesign on the 2nd of February 2023, width=839]

The frame looks strange, I know. I wanted to use a torus because it’s a cool-looking shape, and tentacles because it’s even more cool-looking than a torus.  
It has been designed thanks to link:https://openscad.org/[openSCAD], an … open-source link:https://en.wikipedia.org/wiki/Computer-aided_design[computer-aided design] tool & language (yes, there is such a thing as 3D Design as code), and printed at home on a link:https://eryone.com/fdm/show/1.html[printer] running an open-source firmware, link:https://marlinfw.org/[Marlin].

Should you want to replicate this at home, you can find the link:https://github.com/MerryKombo/3DDesign/tree/MQ-Pro/assets/Booth%20Display[source code] on my link:https://github.com/gounthar[GitHub].
